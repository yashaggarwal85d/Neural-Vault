# Excalidraw Data

## Text Elements
Functional - 

1. Send notifications via Push / Email / SMS
2. Support:
    User-based notifications
    Topic / service-based notifications
3. User preferences (opt-in / opt-out per channel)
4. Template-based messages
5. Deduplication (unique id for each notification)
6. Scheduling (optional but good)
7. Retry (Retry queues)
8. Rate limiting the User (Avoid bombing)
 ^nbZnvO9f

Non-Functional - 

1. 10M notifications/sec peak
2. 99.99% availability
3. Eventual consistency
4. At-least-once delivery
5. Horizontally scalable
5. Low latency for push (<200ms p99) ^REmcTBxZ

Scale - 

1. Peak: 10M notifications/sec
2. Avg payload: 1 KB
3. Write throughput: ~10 GB/sec
4. Active users: 200–300M
5. Fanout ratio:
    70% push
    20% email
    10% SMS ^tKVSkNe6

Fanout Service (Critical)

Two strategies:
1. Fanout-on-write (preferred for push)
    Resolve recipients immediately
    Write per-user notifications to queues

2. Fanout-on-read (for large topics)
    Store event once
    Resolve recipients during delivery

Tradeoff:
Fanout-on-write → faster delivery, higher write cost
Fanout-on-read → storage efficient, higher latency ^HXxJG6wi
