## Text Elements
Functional requirements -

Must-have (Core):

1. System should accept a long URL and generate a unique short URL.
2. Accessing the short URL should redirect (HTTP 301/302) to the original URL.
3. Users should be able to delete or deactivate a short URL they created.

Nice-to-have (Explicitly call these out):

4. Support expiry time (auto-delete after TTL).
5. Support custom aliases (e.g., tiny.url/yash).
6. Track basic analytics (click count, last accessed time) (optional).
 ^ci7M8FHx

Non - Functional requirements -

1. High availability — redirection should never fail (99.9%+).
2. Low latency — redirects should happen within ~50–100ms.
3. Scalability — system should handle:
    a. Millions of URLs
    b.Very high read traffic (read-heavy system)
4. Uniqueness — short URLs must be globally unique.
5. Durability — once created, mappings should not be lost.
6. Eventual consistency acceptable (except during creation).
7. Security & abuse protection — prevent spam, malicious URLs. 

 ^71YL9UqT

Scale - 

1. 10 million total users
2. 10% total active users - 1 million active users
3. 2 URLs per day per user - 2 million writes per day - 23 writes per second - 200 writes per second peak traffic
4. 100x reads than writes - 20k reads per second peak traffic
Read heavy system - 99% reads
High availability more important than strong consistency

Redirect API flow - 
1. User accesses tiny.ly/abc123
2. DNS routes request to nearest data center
3. Load balancer forwards request to a redirect service instance
4. Redirect service: Looks up abc123 in Redis
5. Cache hit: Immediately returns HTTP 301/302 redirect
6. Cache miss:
    a. Fetches mapping from Cassandra
    b. Populates Redis (with TTL)
    c. Returns redirect to user
Latency target: < 100 ms ^jaG4TcRL

Create URL API flow -
1. User submits long URL
2. Request routed via DNS → LB → Create URL service
3. Google Safe Browsing external service for malicious URLs
4. Generates unique ID using Snowflake
5. Encodes ID using Base62 → short URL
6. Stores mapping in Cassandra
7. Optionally stores mapping in Redis
8. Returns short URL to user ^hcmFm8Bx
