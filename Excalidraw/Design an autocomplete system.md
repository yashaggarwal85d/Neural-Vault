# Excalidraw Data

## Text Elements
Functional requirements - 

1. Autocomplete: Suggest top 10 relevant queries based on prefix.
2. Real-time Update: New trending queries should appear within minutes.
3. Personalization: User-specific history takes priority.
4. Filtering: Adult content, hate speech, or sensitive terms must be blocked. ^V86GE6oJ

Non functional requirements - 

1. Ultra-low Latency: 
    Server-side processing should be $< 10ms$ to allow for network overhead
    aiming for $< 100ms$ total UI response.
2. High Availability
3. horizontally scalable
4. Fault tolerance ^432KEjpP

Scale - 

1. 1B DAU x 20 unique queries/day = 20B queries total.
2. Terabytes of data in memory for the Trie ^B1GZfiVp

API design - 

request - GET /v1/suggestions?q={prefix}&limit=5&uid={userId}
Response - {
  "suggestions": [
    {"term": "apple watch", "score": 1200},
    {"term": "apple iphone 15", "score": 950}
  ],
  "latency_ms": 8
} ^zG5VhQNb

Data model - 

The Trie (In-Memory): 
    Store the Trie in a distributed cache like Redis.
Node Optimization: 
    Each node in the Trie shouldn't just store a character; 
    it should store a pre-computed list of the Top 10 suggestions for that prefix. 
    This turns an O(Subtree Search) operation into a simple O(L) lookup.
Permanent Storage (Cassandra): 
    This is the source of truth used to rebuild the Trie periodically. ^A0Fy8Crk
