# Excalidraw Data

## Text Elements
Functional Requirements

1. Metrics ingestion
    Support time-series metrics (counter, gauge, histogram)
    Support push and pull models
2. Data storage
    Store metrics as time-series data
    Support configurable retention (7 days → 1 year)
3. Querying
    Users can query metrics by:
        metric name
        tags/labels
        time range
4. Visualization
    Dashboards with real-time and historical graphs
5. Alerting
    Threshold-based alerts
    Window-based alerts (e.g. avg over 5 min)
6. Integrations
    SDKs / agents / exporters
    HTTP, gRPC ingestion endpoints ^cSMyxZzP

Non-Functional Requirements

1. High availability (no single point of failure)
2. Write optimized
3. Low ingestion latency: < 100 ms from source → storage
4. Query latency:
    Dashboards: < 1–2 sec
    Alerts: near real-time
5. Eventual consistency acceptable
6. Horizontally scalable
7. Multi-tenant isolation ^mnfk3vc3

Scale Assumptions

1. 1 million distinct metrics
2. 10 million data points / second ingestion
3. Retention: 30 days hot, 6 months cold
4. Queries: 100k QPS (dashboards + alerts) ^95yDD4iW

Clients ^wEQMWiwm
