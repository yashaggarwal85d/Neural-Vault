# Excalidraw Data

## Text Elements
Functional requirements -

1. Create a paste with plain text (≤1MB) and optional expiration
2. Generate a unique short URL
3.Retrieve paste via short URL
4. Auto-expire pastes after TTL
5. Optional delete via delete-token
6. optional analytics ^jYv4i3I0

Non - Functional requirements -

1. High availability (no single point of failure)
2. Low read latency (read-heavy system)
3. Durability of pastes until TTL
4. Horizontal scalability
5. Cost efficiency (storage + bandwidth heavy) 

 ^hHdsZ7tg

Scale - 

1. 10 million total users
2. 10% total active users - 1 million active users
3.  10 pastes per user per day = 10 million pastes / day
    Peak = 2× average
    Avg writes/sec =
    10M / 86400 ≈ 116 writes/sec
    Peak ≈ 200–250 writes/sec
4. 10× reads = 100 million reads/day
    Avg reads/sec ≈ 1,150
    Peak ≈ 2,000–2,500 reads/sec
5. Storage needs - 
    max paste size = 1MB
    average paste size = 10KB
    writes per second = 200 = 2000kb per second 
    = 2MB per second = 90000 MB per day = 90GB per day. 
    Assuming we take the storage support for an year! 90*365 = 33 TB per year. 
    ceil approximation to 40 TB per year storage
    taking in replication needs = 100TB per year
Read heavy system - 90% reads
High availability more important than strong consistency

Redirect API flow - 
1. CDN cache check
    If hit → return content immediately
2. Redis cache check
    Lookup paste:{short_url}
    If hit → return content / metadata
3. Metadata DB lookup
    Fetch metadata by short_url
    If not found or expired → 404 / 410
4. Fetch content
    Fetch from object storage using object_key
5. Populate cache
    Store metadata in Redis
    Store content if size ≤ threshold
    Set TTL = expires_at - now
6.Return response
    Content + metadata headers
    CDN cache headers if allowed ^wuzxELZT

Create URL API flow -
1. Rate limit check (write-strict)
    Per IP / per token
    Reject if exceeded (429)
2. Validate request
    Content ≤ 1MB
    UTF-8 plain text
    TTL within allowed range
3. Generate IDs
    Generate paste_id (Snowflake / UUID)
    Generate short_url (Base62)
    Generate delete_token (secure random)
4. Store content
    Write text to Object Storage (S3/GCS)
    Object key = paste_id
5. Store metadata
    Write to Metadata DB (Cassandra/DynamoDB)
    Includes TTL / expires_at
6. Emit expiry event
    Publish (paste_id, expires_at, object_key) to queue
7. Return response
    Return short_url, delete_token, expires_at ^zVGnSZox

3. Delete blob from object storage
