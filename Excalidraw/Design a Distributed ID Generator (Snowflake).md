# Excalidraw Data

## Text Elements
Functional requirements - 

1. Generate globally unique IDs
2. IDs must be roughly time-sortable (monotonic increasing with time)
3. IDs must be generated without coordination between nodes
4. ID generation should be local to the machine (no network call per request)


 ^1yfjE5up

Non-functional requirements - 

1. Very high QPS
    ~10 million IDs/sec globally
    Tens of thousands IDs/sec per node
2. Low latency
    In-memory generation
    p99 < 5 ms
3. Horizontally scalable
    Add/remove nodes without coordination
    Highly available
4. No single point of failure
    System continues working even if nodes go down
5. System must handle clock skew and clock rollback ^FzdGWUjv

Users
Browser/App ^HUR553W3

Load Balancer / API  Gateway (Optional)
Web Application Firewall (WAF)  ^fH7Q6Cx2

ID generator ^RLfkgo0P


Application server ^I4K6aB4L

ID = timestamp | datacenterId | machineId | sequence
          41 bits       5 bits           5 bits      12 bits

timestamp = milliseconds since custom epoch = currentTimeMillis - customEpoch
Datacenter = Small integer (e.g. 0–31)
Machine = Small integer (e.g. 0–31)
Sequence = Small integer (e.g. 0–4095)
Strings are mapped once at startup, not per ID

id =
  (timestamp << (dcBits + machineBits + seqBits)) |
  (dcId << (machineBits + seqBits)) |
  (machineId << seqBits) |
  sequence; ^807HNrot
